반응형 HTML 코딩 가이드

Responsive HTML Coding Guide

Amway 홈페이지의 공지 .os_noticeRes(맞춤메시지, 신제품/프로모션 공지, 일반 페이지 일부 포함), 제품 상세.os_noticeRes.os_prodRes 등 에서 공통으로 사용하는 HTML 가이드

Chrome, Safari, Whale, Firefox, Edge 등 최신 웹/모바일 브라우저 기준

웹표준으로 컨텐츠 작업 후 SLA를 준수한다. W3C Markup 검증 서비스 - https://validator.kldp.org/

코딩 시 기본으로 반응형 공지 클래스를 활용하고, 디자인에 맞춰 기타 인라인 스타일을 추가 사용한다.

반응형 공지 CSS에 클래스를 추가할 땐 반드시 .os_noticeRes 하위에 추가한다.

코딩 시 제목은 h1 ~ h6 대신 p>strong 태그를 사용한다. (Amway 사이트에 h1 ~ h6 에 이미 다른 스타일 적용 되어있음)

<picture> 태그

  • - PC, 모바일 별도 이미지 사용 시 필요한 조건(화면 사이즈)의 이미지만 불러옴

          <source media="(min-width:640px)" srcset="img_pc.png">

        </picture>

PSD → 코딩

Photoshop to HTML/CSS

  • - PSD는 모바일 용으로 기본 2배수로 작업 되어있기 때문에
    PSD → 코딩 시 단위를 1/2 로 해서 작업
  • - ex) PSD 2px 두께의 #000 사각 테두리 → 코딩 CSS에서 border:1px solid #000
  • - ex) PSD 반지름 20px의 둥근 모서리 → 코딩 CSS에서 border-radius:10px
  • - 간격(margin/padding), 컨텐츠 폭(widL 등) , 폰트 크기(fntH1 등) 등 클래스를 정의한 것은 무조건 2배수는 아니고 가이드를 따라간다.
